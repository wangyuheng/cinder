## 新增需求

### 需求: 仪表盘首页
系统应提供仪表盘首页，显示关键指标和最近活动。

#### 场景: 查看仪表盘指标
- **当** 用户打开仪表盘首页
- **则** 系统显示执行次数、成功率、决策次数和文件数量

#### 场景: 查看最近执行
- **当** 用户查看仪表盘
- **则** 系统显示带有状态指示器的最近执行列表

### 需求: 执行页面
系统应提供执行页面，列出所有执行历史。

#### 场景: 查看执行列表
- **当** 用户导航到执行页面
- **则** 系统显示所有执行的分页列表

#### 场景: 按状态筛选执行
- **当** 用户选择状态筛选器
- **则** 系统仅显示匹配所选状态的执行

#### 场景: 查看执行详情
- **当** 用户点击某个执行
- **则** 系统显示详细信息，包括目标、状态、创建的文件和时间戳

### 需求: Soul 配置页面
系统应提供页面来查看和编辑 Soul 配置。

#### 场景: 查看 Soul 配置
- **当** 用户导航到 Soul 页面
- **则** 系统显示当前的 Soul 画像和特质分数

#### 场景: 编辑 Soul 配置
- **当** 用户修改 Soul 特质并保存
- **则** 系统更新 Soul 配置文件

### 需求: 任务触发页面
系统应提供页面来手动触发新的执行任务。

#### 场景: 触发新执行
- **当** 用户输入目标并点击执行
- **则** 系统创建新执行并显示进度

#### 场景: 选择执行模式
- **当** 用户触发新执行
- **则** 系统允许选择 auto、interactive 或 dry-run 模式

### 需求: 决策页面
系统应提供页面来查看决策历史。

#### 场景: 查看决策列表
- **当** 用户导航到决策页面
- **则** 系统显示所有决策的分页列表

#### 场景: 查看决策详情
- **当** 用户点击某个决策
- **则** 系统显示上下文、置信度分数和推理

### 需求: 深色主题支持
系统应支持深色主题作为默认，并提供浅色主题选项。

#### 场景: 默认深色主题
- **当** 用户首次打开仪表盘
- **则** 系统以深色主题显示

#### 场景: 切换主题
- **当** 用户点击主题切换
- **则** 系统在深色和浅色主题之间切换

### 需求: 响应式设计
系统应支持桌面和移动视口。

#### 场景: 移动视口
- **当** 用户在移动设备上访问仪表盘
- **则** 系统显示针对移动端优化的响应式布局
